Representational State Transfer

A rule system for communicating over HTTP.

HTTP Verbs

  GET  

  POST  

  PUT  

  DELETE  

  GET  

Most common HTTP verb

Used for retrieving (read-only) information from the application

  POST  

Second most common HTTP verb

Mostly found in form elements

Used for creating something in the application

  PUT  

Used for editing something in the application

  DELETE  

Used for removing something from the application

Resources

Resources are 'nouns' in your application

  • Studies
  • Programs
  • Parts
  • Records
  • ...etc

REST is about connecting HTTP verbs to resources

Plural and Singular

Resources are thought of in terms of
collections
or
instances

Collections

Refers to the collection of all resources at once.

/studies

/programs

/parts

Instances

Refers to a single resource according to a :resource_id

/studies/:study_id

/programs/:program_id

/parts/:part_id

Collection Examples

VERB /parts
  GET   200(OK) list all parts
  POST   201(CREATED) returns new resource
  PUT   404(NOT FOUND) not possible without an id
  DELETE   404(NOT FOUND) not possible without an id

Instance Examples

VERB /parts/:part_id
  GET   200(OK)
404(NOT FOUND) 		list of all parts
if id not found
  POST   404(NOT FOUND) PUT preferred
  PUT   201(CREATED)
404(NOT FOUND) 		updated record
if id not found
  DELETE   200(OK)
404(NOT FOUND) 		success indicator (true)
if id not found

Succinctness

Resource fetches are short


							[localhost]/api/v2/index.cfm/programs/
						

							[localhost]/api/v2/index.cfm/programs/?program_name=7*7
						

							[localhost]/api/v2/index.cfm/programs/?program_name=7*7&retired=true

Organization

Consideration of resources helps map out the application

Modularity

Data is agnostic of presentation

Selection of different output formats possible

  • JSON
  • XML
  • CSV
  • YAML

Modularity Part II

Data can be reused by clients

Multiple front ends can be serviced by the same application

Multiple use cases:

  • Engineers entering data
  • Managers viewing reports
  • Suppliers reading subset of data

Performance

Data fetches are small and processing is minimal

Application appears faster to users

Reusability

The same code can serve multiple uses
